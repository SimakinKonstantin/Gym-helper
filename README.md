**Система управления программами тренировок**
===

Позволяет составлять план тренировок, запускать тренировки, а также вести историю проведенных тренировок для анализа прогресса.

## Функционал
* Механизм авторизации пользователей;
* Поддержка создания и редактирования программ тренировок с указанием выполненных упражнений, а также характеристик этих упражнений;
* Поддержка запуска выбранных тренировок с отслеживанием прогресса по выполненным упражнениям, а таже времени длительности тренировок;
* Поддержка сохранения статистики о пройденных тренировок, включающих в себя информацию о выполненных упражнениях, а также совету на основе проанализированных данных.

## Особенности
* Авторизация пользователей с помощью JWT;
* Хранения пароля в зашифрованном виде;
* Поддержка мониторинга состояния приложения (Prometheus, Grafana).

## Инструкция по использованию
Выполнить `docker compose up`. Метрики можно увидеть в Grafana, внешний порт которой - 3000. Логин\пароль - admin\admin.

## Архитектура
Приложение использует микросервисную архитектуру. Используемые сервисы:
1. Gateway;
2. Сервис авторизации;
3. Сервис управления тренировками;
4. Сервис расчета статистики.
Подробное описание структуры системы можно увидеть на уровнях C4 диаграммы, описание которых приведено в `./c4`.
После завершения тренировки отправляется Kafka сообщение на анализ данных тренировки, сервис расчета статистики выполняет асинхронный анализ. Это предотвращает зависание потенциально долгого HTTP-запроса от клиента.

## Используемые технологии
* **Backend:** Go;
* **Frontend:** JavaScript, Vue, HTML, CSS;
* **Хранение данных:** PostgreSQL;
* **Брокер сообщений:** Apache Kafka;
* **Мониторинг:** Prometheus, Grafana;
* **Развертывание:** Docker Compose.

## Инструкция по запуску
1. Выполнить: `docker compose up`;
2. Перейти на `localhost:5173`;
3. Для входа в Grafana нужно перейти на `localhost:3000` и использовать логин\пароль `admin`\\`admin`.
