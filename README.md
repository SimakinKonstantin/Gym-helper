**Система управления программами тренировок**
===

Позволяет управлять планом тренировок.

## Функциональные требования
* Механизм авторизации пользователей;
* Функционал создания и редактирования программ тренировок с указанием выполненных упражнений, а также характеристик этих упражнений;
* Функционал запуска выбранных тренировок с отслеживанием прогресса по выполненным упражнениям, а таже времени длительности тренировок;
* Функционал сохранения статистики о пройденных тренировок, включающих в себя информацию о выполненных упражнениях, а также совету на основе проанализированных данных.

## Нефункциональные требования
* Авторизация пользователей с помощью JWT;
* Хранения пароля в зашифрованном виде;
* Поддержка мониторинга состояния приложения (Prometheus, Grafana);
* Развертывание приложения в виде docker compose приложения.

## Инструкция по использованию
Выполнить `docker compose up`

## Архитектура
Приложение использует микросервисную архитектуру. Используемые сервисы:
1. Gateway;
2. Сервис авторизации;
3. Сервис управления тренировками;
4. Сервис рассчета статистики.
Подробное описание структуры системы можно увидеть на уровнях C4 диаграммы, описание которых приведено в `./c4`.
